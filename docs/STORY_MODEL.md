## 제품 원칙 (한 줄)

> **월간 리포트 = 정기 락인 · 챕터 = 드문 서프라이즈 · 실물 책 = 날짜순 아카이브**

---

## 네 가지 레이어

```
일기 (매일)          →  날짜순 한 페이지씩 기록
Story Arc (백그라운드) →  주제별 이야기 줄기, 여러 개 동시 진행
완성 챕터 (앱)       →  한 줄기가 봉인됐을 때 “짜잔” + 리스트
월간 리포트 (앱)     →  그 달 전체 회고, 말일 reveal + 아카이브
실물 책 / PDF        →  선택한 일기를 월·날짜순으로 인쇄 (챕터 틀 없음)
```

| 레이어 | 사용자 인지 | 빈도 | 노출 |
|--------|------------|------|------|
| **일기** | 매일 기록 | 매일 | Feed, 캘린더, 기록 |
| **Story Arc** | 거의 모름 (백그라운드) | 저장마다 | UI 없음 (분류·매핑만) |
| **완성 챕터** | “이야기가 한 권 실렸다” | 가끔 | 앱만 (reveal + 리스트) |
| **월간 리포트** | “이번 달은 이런 달” | 매달 | 앱만 (말일 reveal + 리스트) |
| **실물 책** | “내 다이어리 책” | 원할 때 | PDF·인쇄 |

---

## 비유

| 레이어 | 비유 |
|--------|------|
| 일기 | 매일 한 페이지 |
| Story Arc | 소설의 **서브플롯** (동시에 여러 갈래) |
| 완성 챕터 | 서브플롯 **한 막이 끝남** |
| 월간 리포트 | **그 달 전체 독후감** |
| 실물 책 | **날짜순으로 묶은 원고** |

---

## 매일 주제가 섞여도 괜찮은 이유

예시: 1/1 A · 1/2 A · 1/3 B · 1/4 C · 1/5 B · 1/6 A

- **한 달 = 한 챕터**가 아님 → A·B·C가 **각각 Story Arc**에 매핑됨 (하루 = Arc 1개)
- **완성 챕터**는 한 줄기(A)가 조건 충족 시에만 봉인 — 날짜가 띄엄띄엄이어도 Arc 안에서는 OK
- **주제가 고르게 섞이면** 챕터는 **잘 안 완성**됨 → 그때는 **월간 리포트**가 메인 경험
- **실물 책**은 애초에 **1/1 → 1/2 → … 날짜순**이라 뒤섞이지 않음

### 챕터 완성이 어려운 경우 (현행 규칙)

Gemini 없을 때 특히: `shifting` + 4장, 또는 21일+5장, 또는 AI 봉인 판단 등.  
→ **챕터는 보너스**, **월간은 매달** — 기대치를 이렇게 나눈다.

## 완성 챕터 — 앱 전용

### 보이는 곳 (앱)

| 위치 | 내용 |
|------|------|
| 더보기 → 완성된 챕터 | 목록 |
| 챕터 상세 | 제목, narrative, 무드 타임라인, 대표 사진 |
| `ChapterRevealOverlay` | 완성 시 짜잔 1회 |
| `ChapterWhisperBanner` (Feed) | 진행 중 힌트만 — **완성 전 리스트 없음** |
| 월간 리포트 | `chapterChanges`에 완성 챕터 제목 (문구로만) |

### 안 나가는 곳

| 위치 | 실제 |
|------|------|
| 실물 책 / PDF | **날짜·월 순** 일기만 (`BookPdfPreviewPlanner`) |
| Feed / 캘린더 | 날짜별 일기 (챕터 구분 없음) |

챕터에 속한 **그날 일기**는 실물 책에 **넣을 수** 있으나, **“Chapter 03: ○○”** 구성은 **인쇄본에 없음**.

### 챕터별로 책을 만들면?

챕터 A(1/1,1/2,1/6) → B(1/3,1/5) 순이면 날짜가 **뒤죽박죽** → **현행은 채택하지 않음**.

향후 옵션 후보: 본편 날짜순 + **맨 뒤 챕터 부록** (요약만).

## 추천 UX 전략: 월간 집중 + 챕터 서프라이즈

### 역할 분담

| | 월간 리포트 | 챕터 완성 |
|--|------------|-----------|
| **역할** | 락인·회고·“이번 달 전체” | 서프라이즈·수집·“한 이야기 끝” |
| **빈도** | 매달 1번 (정기) | 가끔 (불규칙) |
| **타이밍** | 말일 reveal (또는 다음 접속) | 기록 저장 후 백그라운드 봉인 |
| **공개 전** | 이번 달 내용 **미노출** | 진행 중 whisper만, **리스트 없음** |
| **공개** | 말일 짜잔 → **월별 아카이브** | 완성 짜잔 → **챕터 리스트** |
| **카피 톤** | “6월의 이야기가 정리됐어요” | “『○○』 이야기가 책에 실렸어요” |

### 왜 이 조합이 최선인가

1. **섞인 일상**에는 월 단위 cross-cutting(월간)이 잘 맞음
2. **챕터는 드물수록** reveal 임팩트가 큼 — “아무도 모르는 사이 완성”
3. **실물 책**은 날짜 아카이브로 역할 분리 유지
4. 챕터 리스트 = **수집감** (책 인쇄와 별개)

### 이미 있는 것 vs 바꿀 것

| | 상태 |
|--|------|
| `ChapterRevealOverlay` | ✅ 유지 |
| `ChaptersScreen` 리스트 | ✅ 유지 |
| `ChapterWhisperBanner` | ✅ 유지 (스포일러 최소) |
| 월간 리포트 말일 reveal + 아카이브 | ❌ [MONTHLY_REVIEW.md](./MONTHLY_REVIEW.md) Phase 1 |
| 더보기 “최근 30일 요약” 상시 노출 | ❌ 제거 예정 |

### 주의

- 챕터가 안 나오는 달이 많아도 OK — **월간이 기대의 중심**, 챕터는 **보너스**
- whisper는 “쓰이고 있어요” 수준 — **완성 전엔 리스트에 올리지 않음** (현행 유지)

## Reveal 두 종류 (톤 분리)

둘 다 `ChapterRevealOverlay` 패턴을 공유하되 **카피·CTA**를 구분한다.

| | `pendingChapterReveal` | `pendingMonthlyReveal` (제안) |
|--|------------------------|-------------------------------|
| 트리거 | Story Arc 봉인 | 말일 / 이전 달 첫 접속 |
| 제목 | 챕터 `displayTitle` | “2026년 6월” |
| CTA | 챕터 상세 | 월간 리포트 상세 |
| 이후 | `ChaptersScreen` | 월간 아카이브 리스트 |

**동시에 둘 다 대기** 시: 우선순위 정책 미정 (챕터 먼저 vs 월간 먼저 vs 큐).

## 데이터 흐름 (요약)

```
[일기 저장]
    → StoryArcEngine (분류·매칭·Arc 상태)
    → (조건 충족) 챕터 봉인 → pendingChapterReveal
    → 일기는 allEntries에 날짜순 유지

[말일 / 다음 접속]
    → 해당 월 집계 → MonthlyReview 생성
    → pendingMonthlyReveal

[실물 책 주문]
    → 사용자가 선택한 DailyEntry[]
    → 날짜순 PDF (ChapterModel 미사용)
```
